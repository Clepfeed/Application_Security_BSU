Генерация сертификата: openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout server.key -out server.crt -subj "/CN=localhost"

- x509 — говорим, что хотим именно сертификат (стандарт X.509), а не просто запрос на него.
- nodes — не ставим пароль на приватный ключ (иначе сервер будет при каждом запуске требовать пароль).
- days 365 — сертификат будет действителен 1 год.
- newkey rsa:2048 — создаем новый приватный ключ алгоритмом RSA длиной 2048 бит.
- keyout и -out — названия файлов, которые появятся.
- subj "/CN=localhost" — "Common Name" (имя домена). Мы указываем localhost.

# Структура:
## Сервер:
В папке Public простой фронтэнд сервера с небольшим скриптом для отправки форм на сервер (задание на осмысленные данные) и свитч правильного/неправильного сертификата (задание на возможность смены на сервере. Проводится по одному клику и отправке запроса на сервер).

В папке Backend хранятся сертификаты, данные с форм и сам код сервера на NODE JS и Express. Отправка сертификатов через TLS для node js.

## Клиент:
В папке certificate находится правильный сертификат сервера. Вся клиентская часть с GUI написана в файле [client.py]("Client/client.py").

Дизайн делал не я, отправил на аутсорс Gemini (в комментариях в коде ниже первый дизайн, решил не удалять на всякий). Работа с сервером происходит с помощью библиотеки requests. Там всё просто, только один запрос с проверкой именно нашего сертификата:

``` Python
response = requests.get(
        SERVER_URL, 
        params={'date': date, 'team': team}, 
        verify=CERT_PATH,
        timeout=5
    )
```

verify=CERT_PATH -- проверка нашего сертификата.
